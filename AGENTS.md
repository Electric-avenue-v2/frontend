# AGENTS.md (Frontend Domain)

## Architecture & Constraints

- **Stack:** Next.js, graphql-request.
- **Data Fetching & Types:** DO NOT manually write TypeScript interfaces for GraphQL queries or mutations. All GraphQL types are auto-generated. Always assume and enforce the use of types generated by running `pnpm run codegen`.
- **Rendering & SEO (High Priority):** SEO is a critical business requirement. You MUST prioritize Static Site Generation (SSG), Incremental Static Regeneration (ISR), or Server-Side Rendering (SSR) for pages and components. Fall back to Client-Side Rendering (CSR) ONLY when dealing with highly interactive states or private user-specific data. Always implement semantic HTML and comprehensive Next.js metadata.
- **Performance Optimization:** Actively anticipate, identify, and fix performance bottlenecks. Prevent unnecessary re-renders, lazy-load heavy assets/components, and ensure code meets Core Web Vitals standards.
- **Styling:** Use STRICTLY plain CSS modules. Do NOT use Tailwind CSS, styled-components, or any utility-first frameworks under any circumstances. Strictly avoid CSS anti-patterns: NEVER use `!important`, avoid overly deep selector nesting, and prevent global scope pollution.
- **Feature-Sliced Design (FSD):** Strictly adhere to FSD principles.
    - `app/`: Global routing, providers, and styles.
    - `app/(pages)/`: Compositional layer.
    - `widgets/`: Standalone, self-contained UI blocks.
    - `features/`: User interactions and business logic chunks.
    - `entities/`: Business entities (e.g., User, Product).
    - `shared/`: Reusable UI components, hooks, utilities, and API instances.
- **Component Design:** Keep UI components dumb. Extract business logic, state management, and side effects into custom hooks.
- Decompose the code as much as possible
- I don't want to see files longer than 200 lines of code.
